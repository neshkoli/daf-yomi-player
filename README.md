## Features

- 📱 **Responsive Design**: Works perfectly on desktop, tablet, and mobile devices
- 🎵 **Advanced Audio Player**: Custom controls with progress tracking and keyboard shortcuts
- 📚 **Smart Navigation**: Easy tractate and daf selection with dropdowns
- ⚡ **Fast Loading**: Optimized for quick loading and smooth playback
- 🔄 **Auto-build**: Automatically scans your content folder and generates the playlist
- 🚀 **GitHub Pages Ready**: Fully static, no server required

## Quick Start

### 1. Setup
```bash
# Install dependencies (optional, for development)
npm install

# Build the data structure from your content folder
npm run build
```

### 2. Development
```bash
# Start local development server
npm run dev
```

### 3. Deploy to GitHub Pages
1. Commit all files to your repository
2. Enable GitHub Pages in repository settings
3. Your player will be available at: `https://yourusername.github.io/repository-name`

## File Structure

```
daf-yomi-player/
├── index.html          # Main HTML file
├── styles.css          # Modern CSS styling
├── script.js           # JavaScript player logic
├── build.js            # Build script to scan content
├── sw.js               # Service worker for caching
├── package.json        # Node.js dependencies
├── data.json           # Generated playlist data
└── content/            # Your audio files
    ├── BavaBatra/
    │   ├── BavaBatra2.mp3
    │   ├── BavaBatra3.mp3
    │   └── ...
    ├── BavaKamma/
    └── ...
```

## Content Organization

Your audio files should be organized as follows:

```
content/
├── TracteName1/
│   ├── TracteName12.mp3
│   ├── TracteName13.mp3
│   └── TracteName1N.mp3
└── TracteName2/
    ├── TracteName22.mp3
    └── ...
```

**Naming Convention**: `{TracteName}{DafNumber}.mp3`

Examples:
- `BavaBatra15.mp3` (Bava Batra, Daf 15)
- `Sanhedrin45.mp3` (Sanhedrin, Daf 45)

## Build Process

The build script (`build.js`) automatically:

1. 🔍 Scans the `content/` folder
2. 📋 Extracts tractate names and daf numbers
3. ✅ Validates file existence
4. 📊 Generates statistics
5. 💾 Creates `data.json` with the playlist structure

### Build Output Example

```json
{
  "BavaBatra": ["2", "3", "4", "5", "..."],
  "BavaKamma": ["2", "3", "4", "5", "..."],
  "Sanhedrin": ["2", "3", "4", "5", "..."]
}
```

## GitHub Pages Deployment

1. **Create Repository**: Create a new GitHub repository for your player

2. **Upload Files**: Commit all files including:
   - `index.html`
   - `styles.css`
   - `script.js`
   - `data.json` (generated by build)
   - `content/` folder with your MP3 files

3. **Enable Pages**: Go to repository Settings → Pages → Source: Deploy from branch → main

4. **Access**: Your player will be available at `https://yourusername.github.io/repository-name`

## Performance Tips

### For Large Collections
- The service worker (`sw.js`) caches static assets
- Audio files are loaded on-demand
- Progress is saved in browser memory during session

### Optimization
- Compress audio files for web (MP3, 64-128 kbps recommended)
- Use consistent naming convention
- Keep file sizes reasonable for mobile users

## Troubleshooting

### Common Issues

**No audio plays**
- Check file paths in browser developer console
- Verify MP3 files are accessible
- Check browser audio permissions

**Missing tractates/dafs**
- Run `npm run build` to regenerate data.json
- Check file naming convention
- Verify content folder structure

**Build script errors**
- Ensure Node.js is installed (v14+)
- Check content folder exists
- Verify file permissions

### Debug Mode
Open browser developer console (F12) to see detailed logs and any errors.
